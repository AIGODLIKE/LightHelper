# LightHelper

[English](README.md) | [简体中文](README_CN.md)

**更直观地控制灯光照亮物体，或根据物体反向管理灯光。**

LightHelper 是一款 Blender 扩展，它将灯光链接（Light Linking）和阴影链接（Shadow Linking）整合为快速、双向的工作流。你既可以从灯光出发，决定它会影响哪些物体；也可以从物体出发，选择哪些灯光要包含或排除该物体。

[下载最新版本](https://github.com/AIGODLIKE/LightHelper/releases/latest) · [反馈问题](https://github.com/AIGODLIKE/LightHelper/issues)

## 功能亮点

- 可以从**灯光**或**物体**任意一侧编辑链接。
- 每盏灯光都能独立切换为**包含**或**排除**模式。
- 可以分别控制**照明**和**阴影**链接。
- 提供专用交互工具，可直接在 3D 视图中编辑链接。
- 自动识别 Blender 灯光物体和使用发光材质的物体。
- 支持灯光独显、EV 能量调节、灯光列表筛选以及工作对象固定。
- 可以手动拆分共享链接数据，也可以选择自动处理灯光副本。
- 一键恢复 Blender 默认的全照明状态。

## 兼容性

| Blender 版本 | Cycles | Eevee |
| --- | :---: | :---: |
| 4.2 | 支持 | 不支持 |
| 4.3 及更高版本 | 支持 | 支持 |

- 最低支持版本为 **Blender 4.2**。
- 使用不支持的渲染器时，灯光链接控制会被禁用。
- LightHelper 的实际效果受 Blender 自身灯光链接能力和渲染器限制影响。

## 安装方法

1. 从[最新版本页面](https://github.com/AIGODLIKE/LightHelper/releases/latest)下载 ZIP 文件，请勿解压。
2. 在 Blender 中打开**编辑 → 偏好设置 → 获取扩展**。不同 Blender 版本中也可能显示为**插件**。
3. 打开右上角菜单，选择**从磁盘安装**。
4. 选择下载的 ZIP；如果 Blender 没有自动启用，请手动启用 **Light Helper**。
5. 在 3D 视图中按 `N`，打开 **LH** 标签页。

侧栏标签默认名为 `LH`，可以在扩展偏好设置中修改。

## 核心概念

### 包含与排除

每盏完成初始化的灯光都有独立的链接模式：

| 模式 | 效果 |
| --- | --- |
| **包含** | 灯光只会照亮链接列表中的物体或集合。 |
| **排除** | 灯光会照亮其他物体，但不会照亮链接列表中的物体或集合。 |

当灯光只需要影响少量指定物体时，适合使用**包含**；当灯光应当正常照亮场景，只需避开少量物体时，适合使用**排除**。

### 照明与阴影

LightHelper 将 Blender 的两种链接通道分别提供出来：

- **照明通道**：控制物体是否接收这盏灯光的照明。
- **阴影通道**：控制物体是否会为这盏灯光投射阴影。

照明和阴影按钮可以独立切换。因此，你可以保留照明但关闭阴影，也可以只调整阴影链接而不改变照明链接。

### 恢复默认全照明

如果想彻底移除一盏灯光的灯光链接和阴影链接集合，请对该灯光使用**还原**。灯光将回到 Blender 默认行为，照亮所有符合条件的物体。

在物体工作流中移除最后一个灯光链接时，受影响的灯光同样会恢复默认全照明状态。

## 交互式灯光链接工具

Blender 处于**物体模式**时，可以在 3D 视图左侧工具栏中找到**灯光链接**工具。它支持直接点选、可移动的快捷键提示以及链接可视化。

### 主体模式

| 主体模式 | 鼠标左键行为 |
| --- | --- |
| **灯光** | 点击灯光可将其设为当前主体；点击物体可切换该物体的链接。 |
| **物体** | 点击物体可将其设为当前主体；点击灯光可切换该灯光与物体之间的链接。 |

按住 `Ctrl` 并点击鼠标左键，可以根据鼠标下的项目切换主体模式。Blender 原生灯光会成为灯光主体，可链接的非灯光物体会成为物体主体。

### 快捷键

| 操作 | 功能 |
| --- | --- |
| 鼠标左键 | 选择新主体，或切换鼠标下项目的两个链接通道。 |
| `Ctrl` + 鼠标左键 | 根据鼠标下的项目切换灯光/物体主体模式。 |
| `L` 或 `Space` | 切换鼠标下项目的照明通道。物体模式下请将鼠标移到灯光上。 |
| `S` | 切换鼠标下项目的阴影通道。物体模式下请将鼠标移到灯光上。 |
| `A` | 在包含和排除之间切换相关灯光。物体模式下请将鼠标移到灯光上。 |
| `X` | 在关闭、选中和全部三种叠加显示模式间循环。 |
| `Ctrl` + 鼠标滚轮 | 切换到筛选结果中的上一个或下一个灯光/物体主体。 |
| 在提示框上按住左键拖动 | 移动快捷键提示框。 |
| `Esc` | 退出并返回之前使用的工具。 |

### 视图叠加显示

默认的**选中**模式只显示当前主体的链接；**全部**模式会显示所有可见链接；**关闭**模式则隐藏链接可视化。

- 绿色表示包含模式的照明链接。
- 红色表示排除模式的照明链接。
- 蓝色表示仅阴影链接。
- 灰色表示两个通道都未启用。

在复杂场景中，LightHelper 会限制物体轮廓的数量，但仍会继续绘制链接线。可以在偏好设置中调整**灯光链接工具最大轮廓数**。

## 侧栏控制

### 灯光链接

主面板会列出可用灯光源及其链接项目。

- 按**全部**、**灯光**或**发光材质**筛选光源。
- 按**全部**、**常规**或**链接中**筛选链接状态。
- 支持搜索、排序、反转可见性，以及隐藏不在当前场景中的光源。
- 初始化或还原单盏灯光、初始化所有灯光，以及清除选中链接。
- 添加物体或集合，并分别切换照明与阴影通道。
- 固定灯光，避免选择变化打断当前设置。
- 让单盏灯光或全部相关灯光使用独立的链接数据。

### 物体链接

物体面板会显示当前选中物体所链接的灯光。

- 直接查看每盏灯光的包含/排除模式。
- 分别切换照明和阴影通道。
- 从物体移除灯光，并在没有剩余链接时安全恢复默认照明。
- 固定当前物体，或显示场景中的已链接物体列表。

### 灯光属性

LightHelper 将相关的 Blender 灯光设置整合到同一侧栏工作流中。对于原生灯光物体，还提供：

- `-1 EV`、`-0.5 EV`、`+0.5 EV` 和 `+1 EV` 能量调节。
- 与当前渲染器对应的 Cycles 或 Eevee 原生灯光设置。
- 选择多个原生灯光时，可同时调整多盏灯光的 EV。

每次 EV 操作都会将灯光能量乘以 `2^EV`。

## 其他功能

### 发光材质检测

材质中存在已连接且强度不为零的发光路径时，使用该材质的物体可以出现在光源列表中。LightHelper 会按照设置的**发光节点搜索深度**检查嵌套节点组。

复杂材质中的搜索深度过高可能会降低列表刷新速度。如果界面出现卡顿，可以适当减小该数值。如果希望在交互工具中将发光物体作为灯光主体，请先从侧栏选择它，或者在工具已经处于灯光模式时用鼠标左键点击它；对非灯光物体使用 `Ctrl` + 鼠标左键，会将其设为物体主体。

独显会在当前筛选出的灯光中仅显示指定的原生灯光。再次点击即可恢复之前的视图可见性、渲染可见性和局部隐藏状态。

### 共享链接数据与灯光副本

复制灯光后，多盏灯光可能会共享 Blender 的链接集合。如果希望每个副本拥有独立链接，可以对单盏灯光使用**设为单用户**，或使用**全部设为单用户**。

**自动修复共享链接**是可选功能，默认关闭。启用后，它只会为明确识别到的灯光副本拆分共享链接集合，并且不会在打开文件时运行。

## 偏好设置

| 设置 | 用途 |
| --- | --- |
| **侧栏名称** | 修改默认的 `LH` 侧栏标签名称。 |
| **发光节点搜索深度** | 设置检测发光材质时搜索嵌套节点组的深度。 |
| **列表筛选类型** | 显示全部光源、仅原生灯光或仅发光材质光源。 |
| **链接筛选类型** | 显示全部光源、未链接光源或已链接光源。 |
| **视图移动类型** | 设置选择时是否移动或以动画方式移动视图。 |
| **自动修复共享链接** | 选择是否为检测到的灯光副本拆分共享链接数据。 |
| **灯光链接工具最大轮廓数** | 限制复杂场景中的物体轮廓数量，链接线仍会显示。 |
| **快捷提示缩放 / HUD X / HUD Y** | 调整交互工具提示框的大小和位置。 |
| **重置灯光链接提示框** | 恢复提示框的默认大小和位置。 |
| **清理旧版残留数据** | 只清理旧版 LightHelper 明确管理的遗留数据。 |

## 已知限制

使用发光材质作为光源时，在 Eevee 中间接光的排除效果可能不稳定。这是 Blender 当前灯光链接实现的限制，而不是 LightHelper 的设置问题。如果需要可预测的排除结果，建议使用 Blender 原生灯光物体。

## 常见问题

### 找不到 `LH` 标签页

- 将鼠标放在 3D 视图上并按 `N`。
- 确认已经在偏好设置中启用 **Light Helper**。
- 检查扩展偏好设置中的**侧栏名称**是否已被修改。

### 无法使用灯光链接工具

- 将 Blender 切换到**物体模式**。
- Blender 4.2 请使用 **Cycles**；Blender 4.3 及更高版本也支持 Eevee。
- 确认当前渲染器支持 Blender 灯光链接。

### 修改一盏复制灯光时，另一盏灯光也发生变化

这些灯光很可能共享同一组链接集合。请使用**设为单用户**，或者启用**自动修复共享链接**，让 LightHelper 自动处理识别到的副本。

### 发光物体没有出现在光源列表中

- 确认材质已启用节点，并存在已连接且强度不为零的发光路径。
- 将光源筛选切换为**全部**或**发光材质**。
- 如果发光节点位于多层嵌套节点组中，请增大**发光节点搜索深度**。

### 灯光没有恢复正常的全照明状态

对该灯光使用**还原**。还原会移除照明接收集合和阴影投射集合，而不是保留一个已经初始化但仍受限制的链接设置。

### 视图中的链接显示过多或影响性能

按 `X` 切换到**选中**或**关闭**模式，或者在偏好设置中降低**灯光链接工具最大轮廓数**。
